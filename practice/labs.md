# Лабораторні роботи: від коду до аналізу трафіку

Лабораторні роботи вимагають встановленого середовища (Python 3.x) та базового розуміння принципів програмування.

## Лабораторна робота 1: Власний TCP-сервер
**Мета:** Зрозуміти, як працює клієнт-серверна архітектура на транспортному рівні (TCP). Ми створимо базовий месенджер (власний Telegram на мінімалках).

**Крок 1: Запуск сервера**
Створіть файл `server.py` і запустіть його. Цей скрипт «слухатиме» порт 65432 на вашому комп'ютері.
```python
import socket

HOST = '127.0.0.1'  # Локальна адреса
PORT = 65432        # Порт для прослуховування

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.bind((HOST, PORT))
    s.listen()
    print(f"Сервер слухає на {HOST}:{PORT}...")
    conn, addr = s.accept()
    with conn:
        print(f"Підключено клієнтом: {addr}")
        while True:
            data = conn.recv(1024)
            if not data:
                break
            print(f"Отримано: {data.decode()}")
            conn.sendall(data) # Відправляємо ехо-відповідь
```

**Крок 2: Запуск клієнта**
Створіть файл `client.py` і запустіть його в іншому вікні терміналу.
```python
import socket

HOST = '127.0.0.1'
PORT = 65432

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.connect((HOST, PORT))
    message = input("Введіть повідомлення для сервера: ")
    s.sendall(message.encode())
    data = s.recv(1024)

print(f"Відповідь від сервера: {data.decode()}")
```
**Завдання для самостійної роботи**. Змініть код так, щоб сервер міг обробляти кілька повідомлень підряд без розриву з'єднання (цикл while), та спробуйте запустити клієнт і сервер на різних комп'ютерах у межах однієї Wi-Fi мережі (змінивши HOST на локальну IP-адресу сервера).

## Лабораторна робота 2: Аналіз мережевого трафіку
**Попередження**: Інструменти перехоплення трафіку (сніфери) використовувати ТІЛЬКИ у власній мережі або з дозволу адміністратора. Незаконне перехоплення даних переслідується законом.
1. Встановіть програму [Wireshark]([url](https://www.wireshark.org/)).
2. Ознайомтесь з інтерфейсом
3. Запустіть перехоплення на вашому активному мережевому інтерфейсі (наприклад, Wi-Fi).
4. У рядку фільтра введіть `http`.
5. Відкрийте браузер і зайдіть на сайт, що не використовує шифрування (наприклад, `http://neverssl.com`).
6. Поверніться у Wireshark і знайдіть пакет з інформацією `GET / HTTP/1.1`.
7. Аналітика: Натисніть на пакет правою кнопкою миші -> Follow -> TCP Stream. Ви побачите весь текст запиту та відповіді. Зробіть висновок, чому вводити логіни та паролі на HTTP-сайтах — це гарантований злив даних.
